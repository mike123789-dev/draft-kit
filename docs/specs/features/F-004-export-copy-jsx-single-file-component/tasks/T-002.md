# T-002: Implement serializeDraftToComponent in core and export from index

## Objective

Add `serializeDraftToComponent(node: DraftNode, registry: ComponentRegistry): string` to `packages/draftkit-core/src/serialize.ts` and re-export it from `packages/draftkit-core/src/index.ts`. The function must traverse the full `DraftNode` tree, collect unique component types, group named imports by `importPath` (sorted alphabetically within each group, groups sorted by path), prepend `import React from "react"`, and wrap the existing `serializeDraftToJsx` output inside `export default function DraftOutput()`.

## Scope

- **Changed files**:
  - `packages/draftkit-core/src/serialize.ts`
  - `packages/draftkit-core/src/index.ts`
- **Dependencies**: T-001 (requires `importPath` field on `ComponentSpec`)

## Validation

- **Required Test Command**: `pnpm --filter @draftkit/core typecheck && pnpm --filter @draftkit/core test`
- **Expected Result**: TypeScript compiles; all existing serialize tests pass; the new function is callable from `@draftkit/core`.

## Implementation Steps

1. Open `packages/draftkit-core/src/serialize.ts`.
2. Add the private helper `collectComponentTypes(node: DraftNode): string[]` that recursively walks `node.children` and accumulates unique type names in a `Set<string>`, returning `[...types]`.
3. Add the exported function `serializeDraftToComponent(node: DraftNode, registry: ComponentRegistry): string`:
   a. Call `collectComponentTypes(node)` to get all unique types.
   b. Build a `Map<string, string[]>` keyed by `importPath`, populated only for types whose registry entry has a non-empty `importPath`.
   c. Construct the output lines array starting with `'import React from "react";'`.
   d. For each `[path, names]` in the map sorted by path, push `import { ${names.sort().join(", ")} } from "${path}";`.
   e. Push an empty line, then `export default function DraftOutput() {`, then `  return (`, then the indented result of `serializeDraftToJsx(node)`, then `  );`, then `}`.
4. Open `packages/draftkit-core/src/index.ts` and add `serializeDraftToComponent` to the named exports from `./serialize`.
5. Run `pnpm --filter @draftkit/core typecheck` — expect 0 errors.
6. Run `pnpm --filter @draftkit/core test` — expect all tests pass.

## Tests / Verification

- Output contains `export default function DraftOutput()`.
- Existing `serializeDraftToJsx` and `serializeDraftToJsxExpression` tests are unaffected.
- No React runtime or framework import is present in `serialize.ts` (file is framework-agnostic).
