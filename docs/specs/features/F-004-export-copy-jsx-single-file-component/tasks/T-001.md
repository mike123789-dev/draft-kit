# T-001: Extend ComponentSpec type and populate importPath on defaultRegistry primitives

## Objective

Add the optional `importPath?: string` field to the `ComponentSpec` type and populate it with `"@draftkit/react"` on all 8 layout primitives in `defaultRegistry`. This unblocks T-002 which needs the registry shape to generate import statements.

## Scope

- **Changed files**:
  - `packages/draftkit-core/src/types.ts`
  - `packages/draftkit-core/src/layout.ts`
- **Dependencies**: none

## Validation

- **Required Test Command**: `pnpm --filter @draftkit/core typecheck && pnpm --filter @draftkit/core test`
- **Expected Result**: TypeScript compiles with no errors; all existing unit tests pass; existing registry entries without `importPath` remain valid (no compile error, no test regression).

## TDD Evidence

### RED

- Command: `pnpm --filter @draftkit/core typecheck`
- Result: Should pass before the change (baseline). Write a test that constructs a `ComponentSpec` with `importPath` and confirm the current type rejects it.

### GREEN

- Command: `pnpm --filter @draftkit/core typecheck && pnpm --filter @draftkit/core test`
- Result: Both commands exit 0 after adding `importPath?: string` to `ComponentSpec` and populating all 8 entries in `defaultRegistry`.

### REFACTOR

- Notes: No structural refactor needed. Verify that no other file constructs `ComponentSpec` objects in a way that would break under the additive change.

## Implementation Steps

1. Open `packages/draftkit-core/src/types.ts`. Locate the `ComponentSpec` type definition.
2. Add `importPath?: string;` as an optional field after the existing `description?: string` field.
3. Open `packages/draftkit-core/src/layout.ts`. Locate `defaultRegistry`.
4. Add `importPath: "@draftkit/react"` to each of the 8 primitive entries: `Page`, `Section`, `Container`, `Stack`, `Inline`, `Grid`, `Spacer`, `Divider`.
5. Run `pnpm --filter @draftkit/core typecheck` — expect 0 errors.
6. Run `pnpm --filter @draftkit/core test` — expect all existing tests to pass.

## Tests / Verification

- TypeScript compiler accepts a `ComponentSpec` constructed with `importPath` present.
- TypeScript compiler accepts a `ComponentSpec` constructed without `importPath` (no regression).
- All 8 primitives in `defaultRegistry` have `importPath === "@draftkit/react"` (assert in a unit test or verify by inspection that T-004 covers this via the serializer output).
- No existing validate, layout, or serialize tests regress.
