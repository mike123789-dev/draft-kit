# T-003: Update DraftKitShell for draft state and copy behavior

## Objective

Wire `DraftNode` state into `DraftKitShell` so that selecting a preset stores the raw `DraftNode`, resetting clears it, and "Copy JSX" uses `serializeDraftToComponent(draft, registry)` when a draft is present — falling back to `editorCode` when not, and disabling the button when both are absent.

## Scope

- **Changed files**:
  - `packages/draftkit-react/src/DraftKitShell.tsx`
- **Dependencies**: T-002 (requires `serializeDraftToComponent` to be exported from `@draftkit/core`)

## Validation

- **Required Test Command**: `pnpm --filter @draftkit/react typecheck && pnpm --filter @draftkit/react test`
- **Expected Result**: TypeScript compiles; shell behavior tests for AC-7 and AC-8 pass; existing shell tests pass.

## TDD Evidence

### RED

- Command: Add a test asserting that after a preset is selected, `navigator.clipboard.writeText` is called with output containing `import React from "react"`, then run `pnpm --filter @draftkit/react test`.
- Result: Should fail before the state and copy-handler changes are made.

### GREEN

- Command: `pnpm --filter @draftkit/react typecheck && pnpm --filter @draftkit/react test`
- Result: Both commands exit 0 after implementing the three changes below.

### REFACTOR

- Notes: Keep the fallback path (`editorCode`) identical in behavior to what existed before this feature. No new abstractions needed.

## Implementation Steps

1. Open `packages/draftkit-react/src/DraftKitShell.tsx`.
2. Import `DraftNode` from `@draftkit/core` if not already imported; import `serializeDraftToComponent` from `@draftkit/core`.
3. Add state: `const [draft, setDraft] = useState<DraftNode | null>(null);`
4. In `handlePreset` (or equivalent preset-select handler): after the call to `createMockDraft`, call `setDraft(result)` where `result` is the returned `DraftNode`.
5. In the reset handler: call `setDraft(null)` alongside any existing reset logic (clearing `editorCode`, etc.).
6. Update `handleCopyCode` (or equivalent):
   - If `draft` is non-null, write `serializeDraftToComponent(draft, registry)` to the clipboard.
   - Else if `editorCode` is non-empty, write `editorCode` to the clipboard (existing fallback, unchanged).
   - Otherwise do nothing (button should already be disabled).
7. Update the "Copy JSX" button's `disabled` prop to `!draft && !editorCode`.
8. Run `pnpm --filter @draftkit/react typecheck` — expect 0 errors.
9. Run `pnpm --filter @draftkit/react test` — expect all tests pass.

## Tests / Verification

- After a preset is selected, `draft` state is non-null (verifiable via mock of `createMockDraft` in `DraftKitShell.test.tsx`).
- Clicking "Copy JSX" after preset select calls `navigator.clipboard.writeText` with the `serializeDraftToComponent` output (string that begins with `import React from "react"`).
- When `draft` is null and `editorCode` is non-empty, "Copy JSX" writes `editorCode` (existing fallback preserved).
- "Copy JSX" button has `disabled` attribute when both `draft` is null and `editorCode` is empty.
- After reset, `draft` is null and the button becomes disabled (assuming `editorCode` is also cleared).
