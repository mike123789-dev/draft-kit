# T-005: Pipeline function and backward compat

## Objective

Wire the pipeline function `validateAndRenderJSX(jsxString, registry)` in `@draftkit/react` that orchestrates core validation + react-live rendering, export `validateJSX` from `@draftkit/core`, and verify that existing `validateDraft()` tests continue to pass unchanged.

## Scope

- **Changed files**: `packages/draftkit-react/src/jsx-renderer.tsx`, `packages/draftkit-react/src/index.ts`, `packages/draftkit-core/src/index.ts`
- **Dependencies**: T-001, T-002, T-003 (validateJSX assembled from parts); T-004 (renderJSX)

Pipeline contract:
```ts
// in @draftkit/react
export function validateAndRenderJSX(
  jsxString: string,
  registry: ComponentRegistry
): { ok: true; element: React.ReactElement } | { ok: false; issues: ValidationIssue[] }
```

The function must not throw for validation failures. `validateJSX(jsxString, registry)` (the pure core function combining parse + forbidden patterns + component names) must also be exported from `@draftkit/core/src/index.ts`. Existing `validateDraft()` export must remain unchanged.

## Validation

- **Required Test Command**: `pnpm test`
- **Expected Result**: Tests pass for: pipeline returns element on valid JSX, pipeline returns issues array on invalid JSX, pipeline never throws, existing validateDraft tests still pass (zero modifications to validate.test.ts).
