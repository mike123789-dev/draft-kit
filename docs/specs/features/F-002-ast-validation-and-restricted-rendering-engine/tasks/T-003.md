# T-003: Registry-scoped component name validation

## Objective

Extend `@draftkit/core/src/jsx-validate.ts` to walk JSXOpeningElement names and reject any component name not present in the provided registry or the implicit layout primitives set.

## Scope

- **Changed files**: `packages/draftkit-core/src/jsx-validate.ts`, `packages/draftkit-core/src/jsx-validate.test.ts`
- **Dependencies**: T-001 (parseJSX types); `@draftkit/core/src/layout.ts` for layout primitives list

Layout primitives always allowed: `Page`, `Section`, `Container`, `Stack`, `Inline`, `Grid`, `Spacer`, `Divider`. HTML elements (lowercase tags) are caught by this check as unrecognized components. Fragment syntax (`<>...</>`) is always allowed and does not produce a JSXOpeningElement name to check.

Export from this task:
```ts
export function checkComponentNames(ast: ParseResult, registry: ComponentRegistry): ValidationIssue[]
```

## Validation

- **Required Test Command**: `pnpm test`
- **Expected Result**: Tests pass for: known registry component accepted, unknown component rejected with name in error, layout primitive accepted without being in registry, lowercase HTML tag rejected, fragment allowed.
