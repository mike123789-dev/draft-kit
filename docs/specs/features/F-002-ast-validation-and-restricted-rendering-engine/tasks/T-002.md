# T-002: Forbidden pattern AST traversal

## Objective

Extend `@draftkit/core/src/jsx-validate.ts` with a data-driven AST traversal that rejects forbidden imports/exports, global identifiers, React hooks, and async patterns — all defined as a `FORBIDDEN_PATTERNS` config structure, not hardcoded branching.

## Scope

- **Changed files**: `packages/draftkit-core/src/jsx-validate.ts`, `packages/draftkit-core/src/jsx-validate.test.ts`
- **Dependencies**: T-001 (parseJSX and ValidationResult types must exist)

The `FORBIDDEN_PATTERNS` data structure must be exported so it can be extended. Each entry specifies the AST node type(s) to detect and the error message template including the offending identifier/keyword. Checks to implement:
- `ImportDeclaration` / `ExportDeclaration` → AC-2
- Identifier references to `window`, `document`, `globalThis`, `eval`, `Function`, `setTimeout`, `setInterval` → AC-3
- Identifier matching hook convention (starts with `use` + uppercase letter, including the named list) → AC-5
- `async`, `await` keywords; `fetch` and `XMLHttpRequest` identifiers → AC-6

Export from this task:
```ts
export function checkForbiddenPatterns(ast: ParseResult): ValidationIssue[]
```

## Validation

- **Required Test Command**: `pnpm test`
- **Expected Result**: Tests pass for each forbidden category: import statement, export statement, each global, hooks (named + pattern), async/await/fetch/XMLHttpRequest. Template literals with forbidden expressions also caught.
