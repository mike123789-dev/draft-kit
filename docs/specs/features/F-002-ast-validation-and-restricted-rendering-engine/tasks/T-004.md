# T-004: react-live rendering engine with scoped component whitelist

## Objective

Implement `renderJSX(code, scope)` in `@draftkit/react/src/jsx-renderer.tsx` using react-live's `LiveProvider` + `LivePreview`, where scope contains only registry-approved components, and missing-component references produce an explicit error instead of rendering undefined.

## Scope

- **Changed files**: `packages/draftkit-react/src/jsx-renderer.tsx` (new), `packages/draftkit-react/src/jsx-renderer.test.tsx` (new)
- **Dependencies**: `react-live` (already installed), `@draftkit/react/src/primitives.tsx` for layout primitives

The scope object is built from: layout primitives (Page, Section, Container, Stack, Inline, Grid, Spacer, Divider) + any components passed in. Before calling LiveProvider, pre-check that all component names referenced in the JSX code string exist as keys in scope — if any are missing, return an explicit error element rather than letting react-live silently render undefined.

Export from this task:
```ts
export function buildScope(registry: ComponentRegistry): Record<string, unknown>
export function renderJSX(code: string, scope: Record<string, unknown>): React.ReactElement
```

## Validation

- **Required Test Command**: `pnpm test`
- **Expected Result**: Tests pass for: valid JSX renders with scoped components, component not in scope produces explicit error message (not undefined/empty), layout primitives available without explicit registry entry.
