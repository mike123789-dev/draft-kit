# T-002: Add buildScopeFromRegistry and update pipeline

## Objective

1. In `jsx-renderer.tsx`: Add `buildScopeFromRegistry(registry: ComponentRegistry): Record<string, unknown>` — iterates registry entries, skips entries with nullish or empty-string `name` or nullish `component`, uses `entry.name` as the scope key, merges with `PRIMITIVE_SCOPE` (registry wins on collision). Keep `buildScope` as an unexported internal function so existing F-002 tests that import it via direct file path continue to work.
2. In `jsx-pipeline.tsx`: Replace `buildScope(additionalScope)` with `buildScopeFromRegistry(registry)`. Remove the `additionalScope?: Record<string, unknown>` parameter from `validateAndRenderJSX`.
3. In `index.ts`: Export `buildScopeFromRegistry`; remove `buildScope` from the barrel export.

## TDD Steps

1. **RED**: Write unit tests for `buildScopeFromRegistry` covering all spec edge cases:
   - Empty registry returns primitives-only scope.
   - Entry with `component: undefined` contributes no scope key.
   - Entry with `component: null` contributes no scope key.
   - Entry with `name: ""` (empty string) is skipped.
   - Entry with valid `name` and `component` adds `{ [name]: component }` to scope.
   - Name collision between two entries — last entry wins (standard object merge).
   - Entry `name` matches a primitive (e.g., `"Stack"`) — registry entry's `component` shadows the primitive.
   - All-primitives-only scope when no entry has a non-nullish `component`.
   Also update or add `validateAndRenderJSX` tests to call with `(jsxCode, registry)` (no third argument).
2. **GREEN**: Implement `buildScopeFromRegistry` in `jsx-renderer.tsx`. Update `validateAndRenderJSX` in `jsx-pipeline.tsx` to use `buildScopeFromRegistry(registry)` and drop `additionalScope`. Update `index.ts` exports.
3. **REFACTOR**: Run `pnpm --filter @draftkit/react test:unit` to confirm all F-002 tests still pass. Confirm `buildScope` is still reachable internally. Confirm `buildScopeFromRegistry` is exported from the barrel.
