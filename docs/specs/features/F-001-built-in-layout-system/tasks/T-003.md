# T-003: Wire layout primitive props in React renderer

## Objective

Update all 8 React layout primitive components to accept and render their declared props from the enriched schema (gap, columns, align, justify, wrap, maxWidth, size, padding, background, etc.), replacing hardcoded defaults with prop-driven values.

## Scope

- **Changed files**: `packages/draftkit-react/src/primitives.tsx`
- **Dependencies**: T-001 (typed prop definitions to know which props each primitive accepts)

## AC Coverage

- **AC-2**: Stack, Inline, Grid accept `gap` prop → controls spacing, default 12
- **AC-3**: Grid accepts `columns` prop → sets grid-template-columns, default 1
- **AC-4**: Stack, Inline accept `align` and `justify` props
- **AC-5**: Inline accepts `wrap` prop (boolean, default true)
- **AC-6**: Container accepts `maxWidth` prop, default 960
- **AC-10**: All 8 primitives render their declared props correctly

## Validation

- **Required Test Command**: `pnpm test`
- **Expected Result**: New tests verify: (1) each primitive renders with correct default styles, (2) passing props changes rendered output (gap, columns, align, justify, wrap, maxWidth, size), (3) unknown props don't leak to DOM.
