# T-002: Add structural validation rules

## Objective

Extend `validateDraft` with structural rules: layout nesting depth limit (4), raw HTML element rejection, children constraint enforcement, and enum/range value validation for props.

## Scope

- **Changed files**: `packages/draftkit-core/src/validate.ts`
- **Dependencies**: T-001 (enriched ComponentSpec with typed props and children constraints)

## AC Coverage

- **AC-7**: Validation rejects drafts with layout nesting depth > 4; depth exactly 4 passes
- **AC-8**: Validation rejects raw HTML elements (div, span, p, h1-h6, ul, ol, li, table, img, a, section, article, header, footer, main, nav, aside, form, input, button, textarea, select, label)

Also validates (from edge cases in spec):
- Spacer/Divider with children → error
- Unknown enum values (e.g., `align: "bogus"`) → error
- Grid columns <= 0 → error
- Negative gap → error

## Validation

- **Required Test Command**: `pnpm test`
- **Expected Result**: All existing validation tests pass. New tests verify: (1) depth 4 passes, depth 5 fails with node path, (2) raw HTML elements rejected at any level with element name in error, (3) children constraint violations caught, (4) enum/range prop values validated.
